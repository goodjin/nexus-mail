# 前端开发计划 - 同步与文件夹

## 文档信息
- **模块编号**: MOD-02
- **模块名称**: 同步与文件夹
- **优先级**: P0

## 1. 模块概述
实现侧边栏文件夹展示、手动刷新、切换账户后的文件夹刷新和列表加载过渡态。

## 2. 技术设计
- 目标文件:
  - `src/components/layout/Sidebar.tsx`
  - `src/hooks/useMailbox.ts`
  - `src/App.tsx`

## 3. 接口调用

| 任务编号 | 接口 | 场景 |
| --- | --- | --- |
| T-01 | `get_folders` | 初始化、切换账户 |
| T-02 | `sync_account` | 手动刷新 |
| T-03 | `get_emails` | 切换文件夹 |

## 4. 开发任务拆分

| 任务编号 | 任务名称 | 文件数 | 行数 | 依赖 |
| --- | --- | --- | --- | --- |

| T-02 | 同步状态与提示条 | 2 | ~100 | T-01 |

## 5. 详细任务定义

### T-01: 文件夹排序与本地化
- **验收标准**:
  - [ ] 系统文件夹顺序符合 PRD
  - [ ] 文件夹名称本地化稳定

### T-02: 同步状态与提示条
- **验收标准**:
  - [ ] 手动刷新有 loading 态
  - [ ] 成功/失败提示明确

### T-03: 切换账户重置策略
- **验收标准**:
  - [ ] 切换账户后 selectedFolder 与 selectedEmail 正确重置
  - [ ] 不残留上一个账户数据

### T-04: 加载态与空态补齐
- **验收标准**:
  - [ ] 请求中显示 loading
  - [ ] 空文件夹显示空态提示
