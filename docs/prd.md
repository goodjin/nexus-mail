# Nexus Mail - 产品需求文档 (PRD)

**版本**: v1.0
**日期**: 2026-03-20
**状态**: 草稿
**对标物**: Apple Mail + Spark Mail
**技术栈**: Tauri (Rust + Web)

---

## 1. 项目概述

### 1.1 背景与目标

**背景**: 市面上缺乏一款轻量、跨平台、界面美观且支持标准协议的开源邮件客户端。Apple Mail 体验优秀但仅限 Apple 生态；Thunderbird 跨平台但界面陈旧；Spark 功能强大但非开源。

**目标**: 打造一款基于 Tauri 的现代化跨平台邮件客户端，聚焦核心收发功能，支持 IMAP/SMTP 标准协议，界面简洁优雅，性能卓越。

**对标参考**:
- Apple Mail：简洁优雅的设计语言、原生级性能
- Spark：现代化的卡片式 UI、智能收件箱分组
- Thunderbird：标准协议支持、开源可扩展

**核心差异**:
| 差异点 | 对标物 | 我们的方案 |
|-------|-------|-----------|
| 技术框架 | Electron (Spark) | Tauri (更小、更快、更安全) |
| 目标定位 | 功能繁多 | 聚焦核心，极简主义 |
| 界面风格 | 传统邮箱样式 | 现代化卡片式设计 |

### 3.2 用户故事

#### US-001: 账户配置

**格式**: 作为用户，我想快速配置邮箱账户，以便开始使用

**详细描述**:
- 用户点击"添加账户"
- 输入邮箱地址，客户端自动识别 IMAP/SMTP 服务器（基于 MX 记录或内置数据库）
- 输入用户名/密码或 OAuth2 认证
- 客户端测试连接并同步文件夹

**验收标准**:
- AC-001-01: 支持自动发现服务器配置
- AC-001-02: 支持 IMAP over SSL (993) 和 STARTTLS (143)
- AC-001-03: 支持 SMTP over SSL (465) 和 STARTTLS (587)
- AC-001-04: 支持 OAuth2 认证（Gmail、Outlook）
- AC-001-05: 连接失败时显示明确错误信息

**边界条件**:
- 输入: 有效的邮箱地址、正确的凭据
- 异常: 无网络、服务器超时、认证失败、SSL 证书错误

---

#### US-002: 邮件列表浏览

**格式**: 作为用户，我想浏览邮件列表，以便快速找到需要的邮件

**详细描述**:
- 左侧显示文件夹列表（收件箱/已发送/草稿等）
- 主区域显示邮件列表，每封邮件显示：发件人、主题、摘要（首行）、时间、附件图标
- 未读邮件高亮显示
- 支持按日期分组（今天/昨天/本周/更早）

**验收标准**:
- AC-002-01: 邮件列表滚动流畅，1000 封邮件无卡顿
- AC-002-02: 未读邮件字体加粗，左侧有未读指示器
- AC-002-03: 鼠标悬停显示邮件预览
- AC-002-04: 支持多选（Shift + 点击、Ctrl + 点击）
- AC-002-05: 批量操作：删除、移动、标记

**对标参考**: Spark 的卡片式布局 vs Apple Mail 的紧凑列表

---

#### US-003: 邮件详情查看

**格式**: 作为用户，我想查看邮件完整内容，以便阅读邮件

**详细描述**:
- 点击邮件后在右侧或新窗口显示详情
- 显示：发件人、收件人（抄送/密送）、主题、时间、附件
- HTML 邮件正常渲染，支持纯文本回退
- 附件可点击下载或内联预览

**验收标准**:
- AC-003-01: HTML 邮件样式正确显示，图片加载正常
- AC-003-02: 支持显示原始邮件头（可展开）
- AC-003-03: 附件列表清晰，支持单文件下载
- AC-003-04: 支持回复/转发/删除操作

---

#### US-004: 撰写发送邮件

**格式**: 作为用户，我想撰写并发送邮件，以便与他人沟通

**详细描述**:
- 点击"撰写"按钮打开编辑器
- 填写收件人（支持自动补全）、抄送、密送
- 输入主题和正文（富文本/纯文本切换）
- 添加附件（拖拽或点击选择）
- 点击发送，邮件进入发件箱并发送

**验收标准**:
- AC-004-01: 收件人自动补全响应 < 200ms
- AC-004-02: 支持附件拖拽添加
- AC-004-03: 草稿自动保存（每 30 秒）
- AC-004-04: 发送失败时提示并保留邮件到草稿
- AC-004-05: 支持 HTML 格式邮件编辑

---

#### US-005: 邮件搜索

**格式**: 作为用户，我想搜索邮件，以便快速定位目标邮件

**详细描述**:
- 顶部搜索框，支持全文搜索
- 搜索范围：主题、发件人、收件人、正文
- 实时显示搜索结果
- 支持搜索过滤器（发件人/日期/附件）

**验收标准**:
- AC-005-01: 搜索响应时间 < 500ms（本地索引）
- AC-005-02: 支持按文件夹范围搜索
- AC-005-03: 搜索历史记录（最近 10 条）
- AC-005-04: 无结果时显示友好提示

---

#### US-006: 邮件组织管理

**格式**: 作为用户，我想组织管理邮件，以便保持收件箱整洁

**详细描述**:
- 将邮件移动到不同文件夹
- 标记星标/旗标
- 标记已读/未读
- 归档邮件
- 删除邮件（移到垃圾箱）

**验收标准**:
- AC-006-01: 支持拖拽移动邮件
- AC-006-02: 右键菜单提供所有操作
- AC-006-03: 批量选择后可批量操作
- AC-006-04: 删除前可配置确认对话框

## 4. 技术需求

### 4.1 技术架构

**框架**: Tauri v2.x
- **后端**: Rust (异步 runtime，使用 tokio)
- **前端**: Web (React/Vue/Svelte 任选)
- **IPC**: Tauri Command (JSON)

**邮件协议支持**:
| 协议 | 支持 | 说明 |
|-----|------|------|
| IMAP4 | P0 | 邮件同步 |
| SMTP | P0 | 邮件发送 |
| POP3 | P1 | 可选支持 |
| EWS | P2 | Exchange 邮件 |

**加密支持**:
- IMAP/SMTP: TLS 1.2+
- 存储: AES-256 本地加密
- 凭据: 系统密钥链集成

### 4.2 性能需求

| 指标 | 要求 | 说明 |
|-----|------|------|
| 冷启动 | < 2 秒 |  |
| 邮件列表加载 | < 500ms | 1000 封邮件 |
| 搜索响应 | < 500ms | 本地索引 |
| 内存占用 | < 200MB | 正常使用 |
| 安装包大小 | < 30MB | Windows |

### 4.3 跨平台支持

| 平台 | 支持 | 优先级 |
|-----|------|--------|
| Windows 10+ | P0 | 高 |
| macOS 11+ | P0 | 高 |
| Linux | P1 | 中 |
| iOS | P2 | 低 |
| Android | P2 | 低 |

## 5. UI/UX 设计方向

### 5.1 设计原则

1. **简洁至上**: 去除一切非必要元素
2. **内容优先**: 邮件内容占据最大空间
3. **一致体验**: 所有平台统一的交互逻辑
4. **性能感知**: 每个操作都有即时反馈
