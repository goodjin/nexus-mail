# Nexus Mail 功能审计报告 (2026-03-23)

本文档梳理了当前 Nexus Mail 客户端支持的操作、实现状态及其核心逻辑。

## 1. 账户管理 (Account Management)

| 操作 | 状态 | 核心逻辑描述 |
| :--- | :--- | :--- |
| **列出所有账户** | ✅ 已实现 | 调用 `list_accounts` 命令，从 SQLite `accounts` 表中检索所有已注册的 Email 地址。 |
| **模拟账户填充** | ✅ 已实现 | `dev_seed_data` 功能。向数据库插入 `demo@nexus-mail.com` 账号并同步初始化加密凭据。 |
| **添加新账户** | ⚠️ 部分实现 | 后端具备 `upsert_account` 能力，但目前缺少前端独立的“添加账号”引导界面。 |
| **删除账户** | ❌ 未实现 | 需要实现清理数据库记录、级联删除相关文件夹/邮件的逻辑。 |

---

## 2. 邮件同步与存储 (Sync & Storage)

| 操作 | 状态 | 核心逻辑描述 |
| :--- | :--- | :--- |
| **文件夹结构同步** | ✅ 已实现 | `sync_account` 时通过 IMAP `LIST` 获取远程文件夹列表并持久化。 |
| **增量邮件拉取** | ✅ 已实现 | `SyncEngine` 逻辑：查询本地 `MAX(uid)` -> 调用 IMAP `UID FETCH` -> 批量写入。 |
| **邮件全文搜索** | ✅ 已实现 | 基于 SQLite FTS5 虚拟表，支持针对主题、发件人和摘要的瞬间检索。 |
| **多账户数据隔离** | ✅ 已实现 | 通过 `account_id` 严格隔离，查询时过滤，确保多账号共存。 |

---

## 3. 邮件核心操作 (Email Operations)

| 操作 | 状态 | 核心逻辑描述 |
| :--- | :--- | :--- |
| **邮件列表显示** | ✅ 已实现 | `get_emails` 命令。支持分页查询，结果按 UID 降序排列。 |
| **发送新邮件** | ✅ 已实现 | `send_email` 命令。使用 `lettre` 库。包含 SMTP 认证与安全投递逻辑。 |
| **拉取详情 (MIME)** | ✅ 已实现 | `get_email_details`。使用 `mailparse` 进行多级 MIME 文本提取（HTML/Plain）。 |
| **附件下载** | ✅ 已实现 | `get_attachment`。支持从多重 MIME 结构中提取指定的原始二进制数据。 |
| **标记已读/未读** | ✅ 已实现 | `update_email_flag` 命令。通过 IMAP `STORE +FLAGS (\Seen)` 指令实现。 |
| **删除邮件** | ✅ 已实现 | `delete_email` 命令。执行 IMAP `STORE +FLAGS (\Deleted)` + `EXPUNGE` 并清理本地 DB。 |

---

## 4. 安全保护 (Security)

| 操作 | 状态 | 核心逻辑描述 |
| :--- | :--- | :--- |
| **凭据加密存储** | ✅ 已实现 | 使用 **AES-256-GCM** 对敏感数据加密存入本地文件，彻底移除 Keychain 依赖。 |
| **主密钥无感派生** | ✅ 已实现 | 基于 **PBKDF2** 结合系统上下文动态派生密钥，实现跨启动周期的无缝解密。 |
