# 模块 PRD：撰写与发送（US-004）

## 目标
保证写信流程清晰、可控，发送失败可恢复，且在多账户场景下明确发件身份。

## 范围
**包含**
1. 撰写器打开/关闭与字段编辑
2. 多账户发件账号显示与切换
3. 附件添加与发送
4. 自动草稿保存与发送失败保护
5. 回复/转发预填

**不包含**
- 邮件模板与签名管理

## 撰写器结构
- 发件账号选择器（多账户必显）
- 收件人/抄送/密送
- 主题输入
- 富文本编辑区（HTML）
- 附件区（拖拽/选择）
 - 收件人与抄送/密送字段可折叠，但状态需可见

## 发送流程
1. 校验必填项（至少 1 个收件人）
2. 点击“发送”后进入发送中状态
3. 发送成功：
   - 关闭撰写器
   - 邮件写入“已发送”
4. 发送失败：
   - 保留内容到草稿
   - 提示失败原因与重试入口

## 草稿与关闭保护
- 自动保存：每 30 秒保存草稿
- 用户关闭或切换页面时：
  - 若内容未发送，弹窗提示“保存草稿/继续编辑/放弃”

## 回复/转发
- 回复：默认收件人为原发件人，主题前缀 “Re: ”
- 转发：主题前缀 “Fwd: ”
- 引用原文（含发件人、时间、主题摘要）
- 发件账号继承当前邮件所属账号

## 校验规则
- 收件人邮箱格式合法
- 附件数量与大小受限（超限时提示）
- 主题可为空但提示用户确认

## 边界条件
- 离线时禁止发送，提示“当前离线，已保存草稿”
- 发送过程中禁止再次点击发送

## 验收标准（AC）
- AC-004-01: 收件人自动补全响应 < 200ms
- AC-004-02: 支持附件拖拽添加
- AC-004-03: 草稿自动保存（每 30 秒）
- AC-004-04: 发送失败时提示并保留邮件到草稿
- AC-004-05: 支持 HTML 格式邮件编辑
- AC-004-06: 存在多个账户时，发送前始终可见当前发件账号
- AC-004-07: 关闭撰写窗口或切换页面时，对未发送内容提供保存草稿保护
