# 模块 PRD：邮件组织管理（US-006）

## 目标
让用户能够快速整理邮箱，同时保持批量操作的简洁与明确。

## 范围
**包含**
1. 标记已读/未读
2. 星标/旗标
3. 归档、删除、移动
4. 拖拽移动
5. 批量操作（仅已读/删除）

**不包含**
- 规则自动归档
- 批量标旗或批量移动 UI（v1 不展示）

## 操作入口
- 列表多选工具栏：仅“标为已读”“删除”
- 详情面板：提供更多操作（回复、转发、删除、标记）
- 右键菜单：可扩展完整操作列表（可选）
- 拖拽：拖到左侧文件夹

## 关键行为
1. 删除操作必须先确认（可通过设置开关控制）
2. 用户点击删除但未确认时，不得删除邮件
3. 删除成功后：
   - 列表移除
   - 多选状态清空
4. 标记已读/未读应立即反馈状态
5. 归档操作将邮件移动到“归档”文件夹
6. 移动操作保留原邮件内容与附件

## 拖拽规则
- 可拖拽单封或多选邮件
- 拖到同一文件夹无效果
- 拖拽完成后显示移动反馈

## 边界条件
- 批量删除失败需回滚列表状态
- 多选状态下保留最后一次显式打开的详情

## 验收标准（AC）
- AC-006-01: 支持拖拽移动邮件
- AC-006-02: 右键菜单提供所有操作
- AC-006-03: 批量选择后可批量操作
- AC-006-04: 删除前可配置确认对话框
- AC-006-05: 多选工具栏不展示移动、旗标、取消、反选等次级按钮，避免干扰主流程
