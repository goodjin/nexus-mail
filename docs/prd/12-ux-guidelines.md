# UI/UX 规范

## 设计原则
1. 简洁至上：去除非必要元素
2. 内容优先：邮件内容占最大空间
3. 一致体验：跨平台统一交互
4. 性能感知：每个操作都有即时反馈

## 信息架构约定
1. 多账户按账户分组浏览，左侧先账户再文件夹
2. 当前上下文继承：列表/详情/回复默认沿用当前账户与文件夹
3. 写信默认发件账号：从当前上下文发起即默认该账号
4. 搜索入口：列表顶部仅保留单一搜索框
5. 账号异常反馈：侧栏与设置页同步显示异常标记
6. 系统文件夹保护：不可重命名/删除

## 关键交互规范
1. 列表顶部仅保留搜索框，不展示范围切换/过滤器/历史
2. 批量工具栏只保留“标为已读”和“删除”
3. 悬停预览仅用于轻量 peek，不替换详情
4. 多选时详情保持最后一次显式打开的邮件
5. 同步状态不占用搜索框附近区域
6. 撰写窗口需明确发件账号，并保护未发送内容

## 组件规范
| 组件 | 样式 | 说明 |
|-----|------|------|
| 邮件卡片 | 圆角卡片，轻阴影 | Spark 风格 |
| 按钮 | 扁平化，主色填充 | Apple 风格 |
| 图标 | SF Symbols / Fluent | 跨平台一致 |
| 字体 | 系统字体 | 原生感 |
| 间距 | 8px 基准网格 | 8/16/24/32 |

## 主题支持
| 主题 | 说明 |
|-----|------|
| 浅色 | 默认主题，适合办公环境 |
| 深色 | 护眼模式，适合夜间使用 |
| 跟随系统 | 自动匹配系统主题 |
