# 模块 PRD：邮件详情查看（US-003）

## 目标
清晰、可靠地展示邮件完整内容与附件，并对隐私与加载失败提供明确反馈。

## 范围
**包含**
1. 正文渲染（HTML/纯文本）
2. 头部信息与附件展示
3. 远程图片策略与单封邮件放行
4. 加载与错误状态

**不包含**
- 邮件编辑或草稿保存
- 搜索高亮

## 详情布局
- 顶部：主题、发件人、收件人（To/CC/BCC）、时间
- 工具栏：回复/转发/删除/标记等快捷入口
- 正文区域：HTML 优先，纯文本回退
- 附件区：列表展示、大小与类型
- 原始头信息：可展开查看 Message-ID、Received 等

## 渲染规则
- HTML 邮件需进行安全渲染与净化
- 无 HTML 时使用纯文本 + 换行处理
- 若正文加载中：显示 Loading 状态
- 加载失败：显示错误原因 + 重试入口

## 远程图片策略
- 默认遵循“隐私设置”：
  - always：自动加载
  - ask：提示并允许“本次加载”
  - never：默认不加载
- 单封邮件临时放行仅作用于当前邮件

## 附件处理
1. 列表展示文件名、大小、类型
2. 支持单文件下载
3. 图片/PDF 内联预览
4. 保存失败时显示明确错误（例如权限不足/路径不可写）

## 边界条件
- 无收件人或抄送：不显示对应字段行
- 邮件过大时延迟渲染，但不阻塞界面
- 附件列表为空时隐藏附件区域

## 验收标准（AC）
- AC-003-01: HTML 邮件样式正确显示，图片加载正常
- AC-003-02: 支持显示原始邮件头（可展开）
- AC-003-03: 附件列表清晰，支持单文件下载，图片/PDF 支持内联预览
- AC-003-04: 支持回复/转发/删除操作
- AC-003-05: 详情加载过程中展示 Loading；失败时展示失败原因
