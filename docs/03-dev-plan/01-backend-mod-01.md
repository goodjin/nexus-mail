## 5. 详细任务定义

### T-01: 账户数据模型校准
- **输出**: `database.rs`, `commands.rs`
- **要求**: 对齐 PRD 中 `Account` 字段语义，明确 TLS/SSL/STARTTLS 与认证方式边界。
- **验收标准**:
  - [ ] 账户记录可表达 IMAP/SMTP 独立安全配置
  - [ ] 支持多账户唯一标识与查询
  - [ ] 不破坏现有命令返回结构

### T-02: 安全存储抽象完善
- **输出**: `security.rs`
- **要求**: 统一账户密码、数据库密钥的读写和错误模型。
- **验收标准**:
  - [ ] 凭据不以明文长期落盘
  - [ ] 获取失败时返回明确错误
  - [ ] 支持后续 OAuth2 token 接入

### T-03: 连接测试链路拆分
- **输出**: `commands.rs`, `imap_client.rs`, `smtp_client.rs`
- **要求**: 将 IMAP/SMTP 测试拆成明确步骤，覆盖认证失败、网络失败、TLS 失败。
- **验收标准**:
  - [ ] 错误信息可区分连接失败与登录失败
  - [ ] 同时覆盖 IMAP 与 SMTP
  - [ ] 满足 AC-001-05

### T-04: 自动发现能力预留接口
- **输出**: `commands.rs`, `database.rs`
- **要求**: 定义服务器自动发现输入输出 DTO 和占位命令，便于后续接入 MX/内置规则库。
- **验收标准**:
  - [ ] 接口定义清晰
  - [ ] 可与前端设置页对接
  - [ ] 与现有手动配置方式兼容

### T-05: 后端测试补齐
- **输出**: 对应 Rust 测试
- **要求**: 覆盖账户 CRUD、密码存取、连接测试错误路径。
- **验收标准**:
  - [ ] 单任务测试用例 ≤ 10
  - [ ] 覆盖多账户隔离
  - [ ] 覆盖安全存储失败场景

## 6. 测试策略
- 单元测试: 安全服务、账户存储、DTO 映射
- 集成测试: IMAP/SMTP 连接测试 mock
