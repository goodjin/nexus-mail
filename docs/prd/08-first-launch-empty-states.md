# 模块 PRD：首次启动与空状态（US-007）

## 目标
在“无账号/无内容/加载失败”时提供清晰引导，避免用户陷入无动作状态。

## 范围
**包含**
1. 无账号空状态
2. 空文件夹/无搜索结果
3. 加载失败状态

## 无账号状态
- 主区域显示欢迎页与主按钮“添加邮箱账户”
- 文案强调下一步操作
- 侧栏按钮仍可见，但不是唯一入口

## 空文件夹状态
- 提示“此文件夹为空”
- 建议“刷新同步”或“切换文件夹”

## 搜索无结果
- 提示“未找到匹配邮件”
- 提供“清空搜索”按钮

## 加载失败
- 提示具体错误原因
- 提供“重试”按钮

## 验收标准（AC）
- AC-007-01: 无账号状态下主操作入口位于主内容区，不仅依赖侧边栏小入口
- AC-007-02: 添加首个账号成功后自动进入可用工作态
- AC-007-03: 空文件夹、空搜索结果、加载失败三类状态均有独立文案和主操作建议
