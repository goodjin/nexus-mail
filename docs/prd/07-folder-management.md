# 模块 PRD：文件夹管理（US-009）

## 目标
提供自定义文件夹的创建、重命名与删除能力，同时保护系统文件夹不被误操作。

## 范围
**包含**
1. 创建自定义文件夹
2. 重命名自定义文件夹
3. 删除自定义文件夹（含确认）
4. 删除当前文件夹后的回退逻辑

**不包含**
- 系统文件夹编辑
- 子层级文件夹结构（v1 禁止“/”层级）

## 操作入口
- 侧栏“New folder”按钮 → 创建
- 自定义文件夹 hover 显示重命名/删除按钮

## 命名规则
- 非空，自动 trim
- 长度上限 64
- 禁止包含 “/” 或 “\”
- 禁止与系统文件夹名称重复（收件箱/已发送/草稿/垃圾箱/垃圾邮件/归档）
- 同账户内不可重复（不区分大小写）

## 创建流程
1. 点击“New folder”打开弹窗
2. 输入名称并校验
3. 创建成功后：
   - 侧栏立即可见
   - 自动选中新文件夹

## 重命名流程
1. 选择自定义文件夹 → 点击重命名
2. 输入新名称并校验
3. 成功后保持当前选中

## 删除流程
1. 点击删除按钮弹出确认
2. 确认后才执行删除
3. 若删除的是当前文件夹：
   - 自动回退到收件箱
4. 删除为服务器侧删除（IMAP delete mailbox），不可在客户端回收

## 错误处理
- 命名冲突：提示“名称已存在”
- 系统文件夹误操作：提示“系统文件夹不可管理”
- 服务端失败：提示具体错误并保持当前状态

## 验收标准（AC）
- AC-009-01: 支持创建自定义文件夹并在侧栏即时可见
- AC-009-02: 支持重命名自定义文件夹并立即同步到侧栏
- AC-009-03: 支持删除自定义文件夹并清理本地缓存
- AC-009-04: 系统文件夹不可被创建/重命名/删除
- AC-009-05: 删除当前文件夹后默认回退到收件箱
