# 前端开发计划 - 账户接入与安全

## 文档信息
- **模块编号**: MOD-01
- **模块名称**: 账户接入与安全
- **对应架构**: `docs/architecture.md#51-账户接入`
- **优先级**: P0

## 1. 模块概述

### 1.1 模块职责
实现账户列表、账户编辑、连接测试和错误反馈，支撑“添加账户”到“完成首次连接”的前端闭环。

### 1.2 页面与组件

| 页面/组件 | 功能 |
| --- | --- |
| `SettingsModal` | 账户创建、编辑、测试 |
| `AccountContext` | 账户列表状态 |
| `Sidebar` | 账户切换 |

## 2. 技术设计
- React + TypeScript
- 目标文件:
  - `src/components/settings/SettingsModal.tsx`
  - `src/context/AccountContext.tsx`
  - `src/components/layout/Sidebar.tsx`

## 5. 详细任务定义

### T-01: 账户表单字段补齐
- **输出**: `SettingsModal.tsx`
- **要求**: 对齐 PRD 的邮箱地址、认证方式、IMAP/SMTP 独立配置。
- **验收标准**:
  - [ ] 字段与 US-001 对齐
  - [ ] 新建与编辑模式分离清晰
  - [ ] 必填项缺失时阻止提交

### T-02: 错误提示与校验态
- **输出**: `SettingsModal.tsx`
- **要求**: 支持连接失败、认证失败、网络异常等用户可理解提示。
- **验收标准**:
  - [ ] 满足 AC-001-05
  - [ ] 有 loading / success / error 三态
  - [ ] 错误不会吞掉

### T-03: 自动发现入口占位
- **输出**: `SettingsModal.tsx`, `AccountContext.tsx`
- **要求**: 保留自动发现触发点和结果回填结构。
- **验收标准**:
  - [ ] UI 有清晰入口
  - [ ] 不影响手动配置
  - [ ] 可接后端发现接口

### T-04: 多账户切换体验
- **输出**: `Sidebar.tsx`, `AccountContext.tsx`
- **要求**: 优化首次默认选中、切换 loading、选中反馈。
- **验收标准**:
  - [ ] 切换账户后视图同步刷新
  - [ ] 选中状态清晰
  - [ ] 与邮箱列表隔离

### T-05: 前端测试补齐
- **输出**: 组件测试 / E2E
- **验收标准**:
  - [ ] 覆盖创建账户成功/失败
  - [ ] 覆盖连接测试失败提示
  - [ ] 覆盖多账户切换

## 6. 测试策略
- 组件测试: SettingsModal 状态流
- E2E: 添加账户 -> 测试连接 -> 保存 -> 切换

## 7. 验收清单
- [ ] 账户管理 UI 闭环完成
- [ ] 错误提示与 loading 态完整
- [ ] 多账户切换体验可用
