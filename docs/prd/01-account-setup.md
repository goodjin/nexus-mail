# 模块 PRD：账户配置与管理（US-001）

## 目标
让用户在首次使用或新增邮箱时，能够快速、可靠地完成账户配置，并在认证/连接异常时获得明确可操作的修复路径。

## 范围
**包含**
1. 新增账户（自动发现 + 手动配置）
2. 认证方式（密码/OAuth2）
3. 连接测试与错误提示
4. 账户异常标记与修复入口
5. 账户基础信息编辑与密码更新

**不包含**
- POP3/EWS 的实现细节（作为后续版本扩展）
- 账号删除后的本地数据清理策略（由“设置中心”模块定义）

## 核心流程
### 1) 首次启动新增账户
1. 空状态主区域展示“添加邮箱账户”主按钮。
2. 用户输入邮箱地址，触发自动发现（基于域名内置配置或 MX 规则）。
3. 自动发现成功：展示推荐 IMAP/SMTP 配置预览。
4. 用户选择认证方式（密码 / OAuth2）并填写凭据。
5. 点击“测试连接”：
   - 成功：允许“保存并同步”。
   - 失败：展示错误原因与可操作建议。
6. 保存成功后：进入该账户收件箱并开始首次同步。

### 2) 自动发现失败 → 手动配置
1. 自动发现失败时展示错误说明和“手动配置”入口。
2. 用户填写 IMAP/SMTP 主机、端口、加密方式。
3. 必须先通过连接测试才允许保存。
4. 保存成功后进入收件箱并同步。

### 3) OAuth2 认证
1. 选择 OAuth2 后引导至浏览器/系统授权。
2. 用户完成授权并返回应用。
3. 授权成功：保存 token 并继续测试连接/同步。
4. 授权失败或取消：保留输入内容，提示重试。

### 4) 账号异常修复
1. 同步/认证异常时，在侧栏账户头像与设置页标红提示。
2. 点击异常提示进入修复面板，展示：
   - 最近错误信息
   - 修复方式（重新认证/修改密码/编辑配置）
3. 修复完成后重新测试并同步。

## UI 结构与字段
**新增账户表单**
- 邮箱地址（必填）
- 显示名称（可选）
- 认证方式（密码/OAuth2）
- 密码（密码模式时必填）
- IMAP 主机、端口、加密方式
- SMTP 主机、端口、加密方式
- 测试连接按钮
- 保存并同步按钮

**状态与反馈**
- 连接中：按钮 loading + 禁用重复操作
- 成功：提示并自动进入收件箱
- 失败：明确错误 + 处理建议

## 校验规则
- 邮箱格式合法（包含 @ 与域名）
- 端口为 1-65535 的整数
- 主机名不得为空
- 认证方式与密码/OAuth2 必须匹配
- 同一邮箱地址不允许重复添加

## 错误处理与提示
- 认证失败：提示“账号或密码错误”，提供“修改密码”入口
- TLS/证书错误：提示“证书校验失败”，提供“检查加密方式/端口”
- 服务器不可达：提示“无法连接服务器”，提供“检查网络/主机地址”
- OAuth2 取消：提示“授权未完成”，允许重试

## 边界条件
- 网络断开时禁止提交，保持输入内容
- 自动发现与手动配置互斥，切换时保留用户已输入内容
- 同一邮箱重复添加需提示并阻止

## 数据与状态
- Account 实体：保存邮箱、服务器、认证方式、同步配置
- 运行状态：last_sync、status、last_error
- 凭据：存储在系统密钥链，不落地明文

## 验收标准（AC）
- AC-001-01: 支持自动发现服务器配置
- AC-001-02: 支持 IMAP over SSL (993) 与 STARTTLS (143)
- AC-001-03: 支持 SMTP over SSL (465) 与 STARTTLS (587)
- AC-001-04: 支持 OAuth2 认证（Gmail、Outlook）
- AC-001-05: 连接失败时显示明确错误信息
- AC-001-06: 账号认证/连接异常时，账号入口显示错误标记
- AC-001-07: 自动发现失败时支持手动输入服务器配置并重新测试
- AC-001-08: 账号异常时支持重新认证、修改密码或编辑服务器配置
