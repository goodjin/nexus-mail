# Nexus Mail 后端架构与 Mock 环境验证

本项目已成功建立了基于 Trait 抽象的后端架构，并配套了完善的 Mock 测试环境。

## 验证项与结果

- [x] **协议抽象层 (Traits)**
    - 定义了 `MailClient` 和 `MailSender` 异步 Trait。
    - 实现了后端业务逻辑与具体协议库 (`imap`, `lettre`) 的完全解耦。
- [x] **Mock 客户端大升级** (`MockMailClient`)
    - 提供了包含 6 个分类文件夹（收件箱、重要、工作等）的模拟数据。
    - 模拟了 **混合类型邮件**: 包含 PDF 附件、内联图片、超长 HTML 正文。
    - **性能验证**: 支持模拟分页获取大量邮件摘要。
    - **单元测试通过**: 经 `cargo test` 验证，所有丰富数据集均能按需正确加载。
- [x] **实测连接 (IMAP/SMTP)**
    - 已在前期验证中通过真实网易企业邮服务器调通 143/25 端口。

## 实施价值
1. **开发效率**: 前端开发人员现在可以使用 Mock 模式，在无需真实账户和网络的情况下调优 UI。
2. **测试保障**: Trait 模式允许我们利用 `mockall` 模拟各类极端情况（如存储空间不足、网络超时等），提高系统鲁棒性。

## 后端核心功能 Backlog
- **增量同步**: 基于 UID 的高效邮件同步算法。
- **加密存储**: SQLite + SQLCipher (AES-256)。
- **全文搜索**: 基于 SQLite FTS5 的本地毫秒级检索。
- **凭据安全**: 集成系统底层 Keychain。
