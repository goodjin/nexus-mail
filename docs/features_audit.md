# Nexus Mail 功能审计与待办清单 (Roadmap)

本文档记录了当前版本已实现的特性，以及后续版本中仍待开发或优化的功能缺口。

## 1. 已实现功能 (Current Features)
- [x] **账户管理**: 多账户添加 (Mock)、列表展示、快速切换。
- [x] **邮件列表**: 多账户隔离加载、未读数实时统计、文件夹切换。
- [x] **全文搜索**: 基于 SQLite FTS5 的快速检索，支持主题与正文匹配。
- [x] **邮件详情**: HTML 安全渲染 (DOMPurify)、元数据展示、附件列表列举。
- [x] **核心操作**: 删除邮件 (本地+服务端同步)、旗标标记、已读/未读切换、存档。
- [x] **写信流**: 支持收件人、主题、正文校验及发送。
- [x] **安全审计**: 全量 E2E XSS 拦截验证。

---

## 2. 待办功能 (Immediate TODOs)

### 2.1 附件深度集成
- [ ] **附件保存**: 目前详情页的 `Download` 按钮为 UI 占位，需对接 `get_attachment` 命令并调用 Tauri 的 `save` 对话框。
- [ ] **写信附件**: 实现写信时的文件选取、大小限制提示及上传逻辑。

### 2.2 交互增强 (UX)
- [ ] **多选操作**: 在邮件列表增加复选框，支持批量“删除”、“标记为已读”或“移动”。
- [ ] **分页/无限滚动**: 目前一次性加载 50-100 封，需补全滚动到底部自动加载 `offset` 的逻辑。
- [ ] **富文本编辑**: 写信框由 `textarea` 升级为支持加粗、链接、列表的编辑器（如 Tiptap 或简单 Markdown）。

### 2.3 状态管理优化
- [ ] **邮件草稿 (Drafts)**: 实现自动保存草稿至本地数据库及服务器同步。
- [ ] **联系人自动补全**: 基于已收/已发邮件地址建立索引，在收件人输入框提供建议。

---

## 3. 后端技术债 (Technical Debt)

### 3.1 同步引擎增强
- [ ] **主动推送 (IMAP IDLE)**: 目前仅支持手动/定时刷新，需评估实现 IDLE 协议以支持实时通知。
- [ ] **SyncEngine 补完**: 补齐 `sync_engine.rs` 中部分 Trait 方法的 `todo!()` 实现，增强单元测试深度。

### 3.2 离线能力
- [ ] **离线操作队列**: 记录用户在断网时的操作 (如删除)，待网络恢复后自动同步至服务端。

---

## 4. 总结
当前 Nexus Mail 已具备 MVP (Minimum Viable Product) 的基础，能够满足日常邮件阅读与简单回复。接下来的重心应放在 **附件的完整闭环** 与 **多选操作效率** 上。
