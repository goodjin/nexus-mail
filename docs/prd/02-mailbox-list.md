# 模块 PRD：邮件列表浏览（US-002）

## 目标
提供高性能、可预期的邮件列表体验，确保浏览、选择和预览不打断当前阅读上下文。

## 范围
**包含**
1. 列表呈现与虚拟滚动
2. 未读/已读/旗标状态显示
3. 悬停预览与多选行为
4. 批量选择后的精简操作栏

**不包含**
- 搜索结果排序算法（由搜索模块定义）
- 邮件详情渲染（由详情模块定义）

## 列表布局与信息层级
- 每封邮件展示：发件人、主题、摘要、时间、附件图标
- 未读邮件：字体加粗 + 右侧未读点
- 旗标邮件：左上角三角标识
- 无主题时显示“(No Subject)”
- 支持按日期分组：今天 / 昨天 / 本周 / 更早

## 交互规则
1. 单击邮件卡片：选中并在详情面板打开
2. 多选触发：Shift+点击、Ctrl/Cmd+点击、或使用列表内选择控件
3. 多选状态：
   - 顶部只显示“标为已读”“删除”
   - 详情面板保持最后一次显式打开的邮件
4. 悬停预览：
   - 仅为轻量提示，不替换详情面板内容

## 性能与加载
- 列表使用虚拟滚动，保证 1000 封邮件滚动流畅
- 首次加载显示“Loading emails...”
- 切换文件夹时显示加载遮罩，避免列表闪烁

## 空状态
- 无账号：显示“添加账号”引导（由空状态模块定义）
- 空文件夹：显示“此文件夹为空”
- 搜索无结果：显示“未找到匹配邮件”
- 加载失败：显示错误原因 + “重试”按钮

## 边界与一致性
- 多选状态下，点击列表空白不清空详情
- 同步或刷新时不改变当前选中的详情内容
- 列表更新时保持滚动位置（同文件夹内）

## 验收标准（AC）
- AC-002-01: 邮件列表滚动流畅，1000 封邮件无卡顿
- AC-002-02: 未读邮件字体加粗，左侧有未读指示器
- AC-002-03: 鼠标悬停显示轻量预览，不替换右侧详情面板内容
- AC-002-04: 支持多选（Shift + 点击、Ctrl + 点击）
- AC-002-05: 批量选择后，顶部操作栏仅保留“标为已读”和“删除”两个主操作
- AC-002-06: 多选状态下不强制切换详情，详情保持最后一次显式点击的邮件
- AC-002-07: 左侧导航清晰展示当前账户与该账户的未读状态
