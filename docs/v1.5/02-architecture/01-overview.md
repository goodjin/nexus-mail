## 1. 系统定位

### 1.1 系统概述
Nexus Mail v1.5 以桌面端邮件客户端为载体，补齐 Spark 级的智能组织与高效处理体验，核心在于“智能收件箱 + 跨账户聚合 + 提醒与计划发送”的闭环。

### 1.2 系统类型
桌面应用（Tauri + React）。

### 1.3 核心功能
- 智能收件箱（分组、优先视图、纠正与指标）
- 跨账户聚合（统一收件箱、全局搜索、过滤）
- 高效处理链路（快速动作、批量一致性、撤销）
- 提醒与跟进（Snooze、未回复提醒、通知）
- 发送与模板增强（计划发送、模板、签名）

### 1.4 目标用户
多账户邮件用户与重度邮件处理用户，关注效率与隐私。

### 1.5 部署方式
本地桌面应用；所有智能能力在本地完成。

---

## 2. 核心设计原则

| 原则 | 说明 |
|---|---|
| 可解释 | 智能分组规则可解释、可被用户纠正 |
| 本地优先 | 所有分类、提醒、调度在本地完成 |
| 上下文一致 | 跨账户视图不丢失账户/文件夹上下文 |
| 可回滚 | 批量操作提供撤销窗口 |
| 轻量安全 | 不引入外部依赖服务或云侧处理 |

## 4. 数据流向图

```
邮件同步 → 本地存储 → 智能分类 → 智能收件箱视图
                   └→ 统一收件箱/全局搜索

用户操作 → 快速动作/批量动作 → 本地状态变更 → 撤销窗口

Snooze/跟进 → 本地调度器 → 到期触发 → 列表置顶 + 通知

计划发送 → 调度器 → SMTP 发送 → 已发送/失败记录
```

## 5. 技术选型

| 类别 | 技术 | 选型理由 |
|---|---|---|
| 前端 | React + TypeScript | 现有前端栈一致 |
| 桌面容器 | Tauri | 轻量与安全 |
| 后端 | Rust | 性能与可靠性 |
| 数据库 | SQLite | 本地存储与多账户隔离 |
| 搜索 | SQLite FTS | 本地全文检索 |
| 通知 | Tauri 通知插件 | 本地桌面通知 |
| 调度 | 本地调度器 | 计划发送与提醒 |
